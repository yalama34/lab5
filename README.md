## Цель проекта
Симуляция казино с игроками разных социальных классов и гусями различных типов

## Структура проекта
```
lab4/
├── src/
│   ├── casino.py        # основной класс симуляции
│   ├── chip.py          # класс фишек с перегрузкой операторов
│   ├── collection.py    # пользовательские коллекции игроков и гусей
│   ├── constants.py     # константы и перечисления
│   ├── goose.py         # классы различных типов гусей
│   ├── main.py          # точка входа
│   └── player.py        # классы игроков разных социальных классов
├── requirements.txt
└── README.md
```

## Запуск
1. Установка зависимостей:
`pip install -r requirements.txt`
2. Запуск симуляции: `python -m src/main`

## Параметры симуляции
При запуске программа запросит:

- Количество шагов симуляции
- Сид для генератора случайных чисел (для воспроизводимости)
- Начальное количество игроков
- Начальное количество гусей

## Типы участников
**Игроки**
- PoorPlayer (бедные): начальный баланс 50 фишек, шанс выигрыша 30%
- MiddlePlayer (средний класс): начальный баланс 100 фишек, шанс выигрыша 50%
- RichPlayer (богатые): начальный баланс 300 фишек, шанс выигрыша 65%

**Гуси**
- BaseGoose: обычный гусь без специальных способностей
- WarGoose: атакует игроков и крадет фишки
- HonkGoose: издает громкие крики
- LeaderGoose: воодушевляет воинов-гусей, увеличивая их силу кражи
- LenderGoose: предлагает игрокам кредиты с процентами
- MimicGoose: имитирует способности других гусей на 3 шага
- TraitorGoose: иногда проявляет милосердие к игрокам

## События симуляции
- Ставки игроков и случайные ставки гусей
- Атаки WarGoose на игроков (богатые игроки имеют шанс избежать атаки)
- Крики HonkGoose и воодушевление воинов LeaderGoose
- Паника бедных игроков с потерей всего баланса
- Выдача кредитов LenderGoose
- Проявление милосердия TraitorGoose
- Имитация способностей MimicGoose
- Появление новых игроков и гусей во время симуляции
- Автоматическое удаление обанкротившихся игроков
- Симуляция завершается при банкротстве всех игроков

## Вывод
Освоены принципы объектно-ориентированного программирования в Python. Реализованы пользовательские коллекции с поддержкой всех необходимых операций, класс с перегрузкой магических методов для работы с фишками казино, система логирования изменений балансов. Симуляция демонстрирует работу различных типов объектов, взаимодействие между ними и динамическое изменение состояния системы во времени
